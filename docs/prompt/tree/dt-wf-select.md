# Step 1: Select Requirement

1. Open `docs/requirements/IMPLEMENTATION_ORDER.md`
2. Find the first unchecked `- [ ]` item
3. Read the full trace:
   - `NORMATIVE.md#PREFIX-NNN` → authoritative statement (MUST/SHOULD/MAY)
   - `specs/PREFIX-NNN.md` → detailed specification, acceptance criteria, test plan
   - `VERIFICATION.md` → how to verify
   - `TRACKING.yaml` → current status (should be `gap`)
4. Read the relevant SPEC.md section(s) cited in the spec file
5. Understand the requirement fully before proceeding

**Gate:** You must be able to explain what the requirement does, why it exists, and how to test it before moving to Step 2.
