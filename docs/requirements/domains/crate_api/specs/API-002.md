## Summary

`CoinRecord` is the core data type representing the full lifecycle state of a coin in the coinstate. It tracks when a coin was created, when (if ever) it was spent, whether it is a coinbase reward, and whether it is eligible for singleton fast-forward.

This corresponds to Chia's `CoinRecord` ([`coin_store.py:46-56`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/coin_store.py#L46)), with the same fields adapted for Rust types. Key difference: Chia uses `spent_index = 0` for unspent and `-1` for FF-eligible unspent; dig-coinstore uses `Option<u64>` for `spent_height` and a separate `ff_eligible` boolean for clarity.

Once a coin record is created, it persists permanently in the coinstate (even after spending) to support historical queries and rollback.

## Specification

### Struct Definition

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct CoinRecord {
    /// The coin (parent_coin_info, puzzle_hash, amount). Immutable once created.
    pub coin: Coin,
    /// Block height when this coin was created (confirmed).
    /// Chia: `confirmed_index` in coin_record table (coin_store.py:49).
    pub confirmed_height: u64,
    /// Block height when this coin was spent. `None` if unspent.
    /// Chia: `spent_index` (coin_store.py:50). Chia uses 0 for unspent,
    /// -1 for FF-eligible unspent; dig-coinstore uses Option<u64>.
    pub spent_height: Option<u64>,
    /// Whether the coin is a coinbase reward (block reward, not a transaction output).
    /// Chia: `coinbase` (coin_store.py:51).
    pub coinbase: bool,
    /// Timestamp of the block that created this coin.
    /// Chia: `timestamp` (coin_store.py:55).
    pub timestamp: u64,
    /// Whether this coin is a potential singleton fast-forward candidate.
    /// Set to `true` when `same_as_parent=true` at creation time (same puzzle_hash
    /// and amount as parent, non-coinbase). Recomputed during rollback.
    /// Chia equivalent: `spent_index = -1` (coin_store.py:128-129).
    pub ff_eligible: bool,
}
```

### Derived Methods

```rust
impl CoinRecord {
    /// Create a new unspent coin record.
    pub fn new(coin: Coin, confirmed_height: u64, timestamp: u64, coinbase: bool) -> Self;

    /// Check if the coin has been spent.
    pub fn is_spent(&self) -> bool;

    /// Mark the coin as spent at the given height.
    pub fn spend(&mut self, height: u64);

    /// Get the coin ID (derived from coin fields).
    pub fn coin_id(&self) -> CoinId;

    /// Convert to a lightweight CoinState for sync protocol.
    pub fn to_coin_state(&self) -> CoinState;

    /// Convert from a `chia-protocol::CoinRecord` for interop.
    pub fn from_chia_coin_record(record: chia_protocol::CoinRecord) -> Self;

    /// Convert to a `chia-protocol::CoinRecord` for interop.
    pub fn to_chia_coin_record(&self) -> chia_protocol::CoinRecord;
}
```

### Method Semantics

| Method | Behavior |
|--------|----------|
| `new()` | Creates record with `spent_height: None`, `ff_eligible: false`. |
| `is_spent()` | Returns `self.spent_height.is_some()`. |
| `spend(height)` | Sets `self.spent_height = Some(height)`. |
| `coin_id()` | Delegates to `self.coin.coin_id()` which computes `sha256(parent \|\| puzzle_hash \|\| amount)`. |
| `to_coin_state()` | Returns `CoinState { coin, created_height: Some(confirmed_height as u32), spent_height: spent_height.map(\|h\| h as u32) }`. |
| `from_chia_coin_record()` | Converts `chia_protocol::CoinRecord` to dig-coinstore `CoinRecord`. Heights are widened from `u32` to `u64`. `spent_block_index == 0` maps to `spent_height = None`. `ff_eligible` defaults to `false` (not present in Chia's type). |
| `to_chia_coin_record()` | Converts to `chia_protocol::CoinRecord`. Heights narrowed from `u64` to `u32` (panics if `> u32::MAX`). `spent_height = None` maps to `spent_block_index = 0`. `ff_eligible` is lost. |

### Design Decision: Custom CoinRecord vs chia-protocol::CoinRecord

dig-coinstore defines its own `CoinRecord` instead of using `chia_protocol::CoinRecord` for three reasons:

1. **Height width**: `chia_protocol::CoinRecord` uses `u32` heights (`confirmed_block_index`, `spent_block_index`). dig-coinstore uses `u64` to future-proof beyond 4 billion blocks.
2. **Option semantics**: `chia_protocol::CoinRecord` uses `spent_block_index = 0` as a sentinel for "unspent". dig-coinstore uses `Option<u64>` which is idiomatic Rust and eliminates sentinel value ambiguity.
3. **FF-eligible tracking**: dig-coinstore adds `ff_eligible: bool` for singleton fast-forward support. This field does not exist in any Chia crate.

`chia_sdk_coinset::CoinRecord` was also evaluated but is an RPC client response type with a redundant `spent: bool` field and serde JSON annotations, unsuitable for a storage layer.

Interop conversion methods are provided for integration with systems that use `chia_protocol::CoinRecord`.

## Implementation Notes

- **Primary file:** `src/types.rs` or `src/coin_record.rs`
- **Type imports:** `Coin`, `Bytes32` from `chia-protocol`. `CoinState` from `chia-protocol`.
- **CoinId alias:** `type CoinId = Bytes32` for semantic clarity.
- **ff_eligible default:** `new()` defaults to `false`. Set to `true` by `apply_block()` when `CoinAddition::same_as_parent` is `true` and the coin is not coinbase.
- **Rollback:** During rollback, `ff_eligible` is recomputed by checking if the coin's parent has the same puzzle_hash and amount and is itself spent (Chia: [`coin_store.py:598-623`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/coin_store.py#L598)).
- **u32 vs u64 in CoinState:** Chia's `CoinState` uses `Option<u32>` for heights. The `to_coin_state()` method casts u64 to u32; this is safe for practical blockchain heights but should debug-assert no truncation.
