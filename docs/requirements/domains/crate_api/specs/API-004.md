## Summary

`CoinStoreError` is the error type returned by all fallible coinstate operations (`new()`, `apply_block()`, `rollback_to_block()`, queries, snapshots, etc.). It is a comprehensive enum covering all possible failure modes: chain continuity violations, coin existence checks, spend validity, storage layer errors, serialization issues, block structure validation, genesis state, and query parameter validation.

The enum derives `Debug + Clone + PartialEq` for testability. `thiserror::Error` provides `Display` and `std::error::Error` trait implementations. Storage backend errors (from LMDB/RocksDB) and serialization errors (from bincode/serde) are stored as `String` to satisfy `Clone + PartialEq` bounds.

Chia uses exceptions (`ValueError`, assertion failures) for equivalent error conditions. dig-coinstore uses typed Rust enum variants with structured data for better ergonomics and pattern matching.

## Specification

### Enum Definition

```rust
#[derive(Debug, Clone, PartialEq, thiserror::Error)]
pub enum CoinStoreError {
    // -- Chain continuity --
    #[error("block height {got} does not follow current height {expected}")]
    HeightMismatch { expected: u64, got: u64 },

    #[error("parent hash mismatch: expected {expected}, got {got}")]
    ParentHashMismatch { expected: Bytes32, got: Bytes32 },

    #[error("state root mismatch: expected {expected}, computed {computed}")]
    StateRootMismatch { expected: Bytes32, computed: Bytes32 },

    // -- Coin existence --
    #[error("coin not found: {0}")]
    CoinNotFound(CoinId),

    #[error("coin already exists: {0}")]
    CoinAlreadyExists(CoinId),

    // -- Spend validity --
    #[error("double spend: coin {0} already spent")]
    DoubleSpend(CoinId),

    #[error("spend count mismatch: expected {expected} updates, got {actual}")]
    SpendCountMismatch { expected: usize, actual: usize },

    // -- Block structure --
    #[error("invalid reward coin count: expected {expected}, got {got}")]
    InvalidRewardCoinCount { expected: String, got: usize },

    #[error("hint too long: {length} bytes exceeds maximum {max}")]
    HintTooLong { length: usize, max: usize },

    // -- Genesis --
    #[error("genesis already initialized")]
    GenesisAlreadyInitialized,

    #[error("coinstate not initialized (call init_genesis first)")]
    NotInitialized,

    // -- Query --
    #[error("puzzle hash batch size {size} exceeds maximum {max}")]
    PuzzleHashBatchTooLarge { size: usize, max: usize },

    // -- Storage --
    #[error("storage error: {0}")]
    StorageError(String),

    #[error("serialization error: {0}")]
    SerializationError(String),

    #[error("deserialization error: {0}")]
    DeserializationError(String),
}
```

### Variant Count

15 variants total covering all coinstate failure modes.

### Variant Categories

| Category | Variants | Triggered By |
|----------|----------|-------------|
| Chain continuity | HeightMismatch, ParentHashMismatch, StateRootMismatch | `apply_block()` validation |
| Coin existence | CoinNotFound, CoinAlreadyExists | `apply_block()` removal/addition validation |
| Spend validity | DoubleSpend, SpendCountMismatch | `apply_block()` spend operations |
| Block structure | InvalidRewardCoinCount, HintTooLong | `apply_block()` structural validation |
| Genesis | GenesisAlreadyInitialized, NotInitialized | `init_genesis()`, pre-genesis operations |
| Query | PuzzleHashBatchTooLarge | `batch_coin_states_by_puzzle_hashes()` |
| Storage | StorageError, SerializationError, DeserializationError | Backend I/O, serde operations |

### Derive Requirements

- `Debug`: Standard error formatting and logging.
- `Clone`: Required for PartialEq and for returning errors from batch operations.
- `PartialEq`: Required for test assertions (`assert_eq!`).
- `thiserror::Error`: Provides `Display` and `Error` trait implementations.

### Storage/Serialization Error Conversion

```rust
impl From<heed::Error> for CoinStoreError {
    fn from(e: heed::Error) -> Self {
        CoinStoreError::StorageError(e.to_string())
    }
}

impl From<rocksdb::Error> for CoinStoreError {
    fn from(e: rocksdb::Error) -> Self {
        CoinStoreError::StorageError(e.to_string())
    }
}

impl From<bincode::Error> for CoinStoreError {
    fn from(e: bincode::Error) -> Self {
        CoinStoreError::SerializationError(e.to_string())
    }
}
```

## Implementation Notes

- **Primary file:** `src/error.rs`
- **thiserror:** Add `thiserror` to `Cargo.toml` dependencies.
- **Bytes32 Display:** `Bytes32` must implement `Display` for the `#[error(...)]` format strings. If not, use `{:?}` or hex-encode.
- **InvalidRewardCoinCount.expected:** Uses `String` rather than a numeric type because the expected count varies: "0" for genesis, ">= 2" for non-genesis blocks.
- **SpendCountMismatch:** Adopted from Chia's strict spend count assertion ([`coin_store.py:645-648`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/coin_store.py#L645)). This is a defense-in-depth check at the storage layer.
