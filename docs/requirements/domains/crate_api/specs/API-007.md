## Summary

`CoinStoreStats` provides a comprehensive snapshot of the current coinstate, returned by `CoinStore::stats()`. It aggregates key metrics: chain tip height and timestamp, coin counts (unspent and spent), total unspent value, the current state root and tip hash, hint count, and snapshot count.

The struct is designed for monitoring, logging, and operational decisions. Unlike Chia (which requires multiple separate queries), dig-coinstore provides all statistics in a single call backed by materialized aggregate counters (SPEC Section 1.6 Improvement #18).

## Specification

### Struct Definition

```rust
pub struct CoinStoreStats {
    /// Current chain tip height.
    pub height: u64,
    /// Current timestamp.
    pub timestamp: u64,
    /// Number of unspent coins.
    pub unspent_count: u64,
    /// Number of spent coins (in history).
    pub spent_count: u64,
    /// Total value across all unspent coins.
    pub total_unspent_value: u64,
    /// Current state root.
    pub state_root: Bytes32,
    /// Current chain tip hash.
    pub tip_hash: Bytes32,
    /// Number of hints stored.
    pub hint_count: u64,
    /// Number of snapshots stored.
    pub snapshot_count: usize,
}
```

### Method Signature

```rust
impl CoinStore {
    /// Get comprehensive statistics about the coinstate.
    pub fn stats(&self) -> CoinStoreStats;
}
```

### Field Sources

| Field | Source | Description |
|-------|--------|-------------|
| `height` | Chain metadata | Current tip height (0 after genesis) |
| `timestamp` | Chain metadata | Timestamp of current tip block |
| `unspent_count` | Materialized counter | Running count updated on insert/spend |
| `spent_count` | Materialized counter | Running count updated on spend |
| `total_unspent_value` | Materialized counter | Running sum of unspent coin amounts |
| `state_root` | Merkle tree | Current root hash |
| `tip_hash` | Chain metadata | Hash of current tip block |
| `hint_count` | Hint store | Total hints in the hint index |
| `snapshot_count` | Snapshot store | Number of retained snapshots |

### Performance

`stats()` SHOULD return in O(1) time by reading pre-computed materialized counters. All aggregate fields (`unspent_count`, `spent_count`, `total_unspent_value`, `hint_count`) are maintained as running values updated atomically during `apply_block()` and `rollback_to_block()` (SPEC Section 1.6 Improvement #18).

## Implementation Notes

- **Primary file:** `src/types.rs` and `src/store.rs`
- **Materialized counters:** Maintain `unspent_count`, `spent_count`, `total_unspent_value`, `hint_count` as running values in chain metadata. Updated atomically with each block application:
  - `apply_block`: `unspent_count += additions_count + coinbase_count - removals_count`, `spent_count += removals_count`, `total_unspent_value += added_value - removed_value`.
  - `rollback_to_block`: counters recomputed from the modified_coins or reverted by subtracting the rolled-back block's deltas.
- **snapshot_count:** Read from the snapshot store's list of available heights.
- **hint_count:** Maintained as a running counter, incremented when hints are stored during block application.
- **Derives:** Consider `Clone` and `Debug` for ergonomic use.
