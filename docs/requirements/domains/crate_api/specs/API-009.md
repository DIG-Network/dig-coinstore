# API-009: Type Aliases and Lineage Info

## Summary

The crate defines and publicly exports semantic type aliases `CoinId = Bytes32` and `PuzzleHash = Bytes32`. These aliases provide clarity throughout the API surface, making function signatures self-documenting (e.g., `fn get_coin_record(&self, coin_id: &CoinId)` is clearer than `fn get_coin_record(&self, coin_id: &Bytes32)`).

Additionally, the `UnspentLineageInfo` struct captures the lineage chain needed for singleton fast-forward resolution: the coin's own ID, its parent's ID, and its parent's parent's ID.

---

## Specification

### Type Aliases

```rust
/// A 32-byte coin identifier (SHA-256 hash of parent_coin_info || puzzle_hash || amount).
pub type CoinId = Bytes32;

/// A 32-byte puzzle hash.
pub type PuzzleHash = Bytes32;
```

### UnspentLineageInfo Struct

```rust
/// Lineage information for an unspent coin, used for singleton fast-forward resolution.
#[derive(Debug, Clone, PartialEq)]
pub struct UnspentLineageInfo {
    /// The coin's own ID.
    pub coin_id: CoinId,
    /// The coin's parent ID (parent_coin_info from the Coin struct).
    pub parent_id: CoinId,
    /// The coin's grandparent ID (parent_coin_info of the parent coin).
    pub parent_parent_id: CoinId,
}
```

### Consistency Requirement

The `CoinId` and `PuzzleHash` type aliases MUST be used consistently throughout the public API surface. This includes:
- All method signatures on `CoinStore`
- All struct field types in `CoinRecord`, `BlockData`, `CoinAddition`, `ApplyBlockResult`, `RollbackResult`, `CoinStoreSnapshot`
- All error variant associated data in `CoinStoreError`

## Acceptance Criteria

- [x] `CoinId` is publicly exported as `type CoinId = Bytes32`.
- [x] `PuzzleHash` is publicly exported as `type PuzzleHash = Bytes32`.
- [x] `UnspentLineageInfo` is publicly exported with fields: `coin_id`, `parent_id`, `parent_parent_id`.
- [x] `UnspentLineageInfo` derives `Debug`, `Clone`, `PartialEq`.
- [x] `CoinId` alias is used consistently in all public method signatures and struct fields.
- [x] `PuzzleHash` alias is used consistently in all puzzle-hash-related signatures and fields.

> **Consistency note:** `CoinStore` accessors that return chain header or Merkle roots (`tip_hash`, `state_root`)
> intentionally remain [`Bytes32`] because those hashes are not coin IDs; `tests/api_009_tests.rs` documents
> the alias surface on the wire types listed in this spec’s “Consistency Requirement” section.

## Implementation Notes

- **Primary file:** `src/types.rs`
- **Dependencies:** `Bytes32` from `chia-protocol`
- **Edge cases:**
  - `CoinId` and `PuzzleHash` are the same underlying type (`Bytes32`); the Rust type system does not prevent mixing them. Developers must rely on naming conventions for correctness.
  - `UnspentLineageInfo` for a genesis coin: `parent_id` is the coinbase parent info, `parent_parent_id` may not exist (use a zero hash or Option).
