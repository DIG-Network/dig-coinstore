## Summary

`ApplyBlockResult` is the success return type of `CoinStore::apply_block()`. It provides a summary of the block's impact on the coinstate: the new state root, counts of coins created and spent, and the new chain tip height.

`RollbackResult` is the success return type of `CoinStore::rollback_to_block()` and `CoinStore::rollback_n_blocks()`. It provides the set of modified coin records (both deleted and un-spent), counts of each, and the new chain tip height.

Chia's `CoinStore.rollback_to_block()` returns `dict[bytes32, CoinRecord]` ([`coin_store.py:567`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/coin_store.py#L567)). dig-coinstore enriches this with explicit `coins_deleted` and `coins_unspent` counts (SPEC Section 1.6 Improvement #11).

## Specification

### ApplyBlockResult Struct Definition

```rust
/// Result of applying a block to the coinstate.
pub struct ApplyBlockResult {
    /// The new state root after applying this block.
    pub state_root: Bytes32,
    /// Number of coins created (additions + coinbase).
    pub coins_created: usize,
    /// Number of coins spent (removals).
    pub coins_spent: usize,
    /// New chain tip height.
    pub height: u64,
}
```

### RollbackResult Struct Definition

```rust
/// Result of a rollback operation.
pub struct RollbackResult {
    /// Coin records that were modified during rollback.
    /// Includes both deleted coins (created after target) and
    /// un-spent coins (spent after target).
    /// Chia returns this as `dict[bytes32, CoinRecord]` (coin_store.py:567).
    pub modified_coins: HashMap<CoinId, CoinRecord>,
    /// Number of coins deleted (created after rollback height).
    pub coins_deleted: usize,
    /// Number of coins un-spent (spent after rollback height).
    pub coins_unspent: usize,
    /// New chain tip height after rollback.
    pub new_height: u64,
}
```

### Field Sources

**ApplyBlockResult:**

| Field | Source | Description |
|-------|--------|-------------|
| `state_root` | Merkle tree recomputation | Root hash after all additions/removals |
| `coins_created` | `block.additions.len() + block.coinbase_coins.len()` | Total coins inserted |
| `coins_spent` | `block.removals.len()` | Total coins marked spent |
| `height` | `block.height` | New chain tip |

**RollbackResult:**

| Field | Source | Description |
|-------|--------|-------------|
| `modified_coins` | Rollback scan | Coins created after target (deleted) + coins spent after target (un-spent) |
| `coins_deleted` | Count of coins with `confirmed_height > target` | Coins removed from the store |
| `coins_unspent` | Count of coins with `spent_height > target` | Coins reverted to unspent |
| `new_height` | `target_height` | New chain tip after rollback |

### Return Context

```rust
impl CoinStore {
    pub fn apply_block(&mut self, block: BlockData) -> Result<ApplyBlockResult, CoinStoreError>;
    pub fn rollback_to_block(&mut self, target_height: i64) -> Result<RollbackResult, CoinStoreError>;
    pub fn rollback_n_blocks(&mut self, n: u64) -> Result<RollbackResult, CoinStoreError>;
}
```

## Acceptance Criteria

- [ ] `ApplyBlockResult` is publicly exported from the crate.
- [ ] All 4 `ApplyBlockResult` fields are present and public.
- [ ] `RollbackResult` is publicly exported from the crate.
- [ ] All 4 `RollbackResult` fields are present and public.
- [ ] `modified_coins` type is `HashMap<CoinId, CoinRecord>`.
- [ ] `state_root` type is `Bytes32`.
- [ ] `apply_block()` returns `Result<ApplyBlockResult, CoinStoreError>`.
- [ ] `rollback_to_block()` returns `Result<RollbackResult, CoinStoreError>`.
- [ ] `coins_created` equals additions + coinbase count.
- [ ] `coins_deleted` + `coins_unspent` equals `modified_coins.len()` (or their sum reflects the rollback scope).

## Implementation Notes

- **Primary file:** `src/types.rs` or `src/results.rs`
- **Type imports:** `Bytes32` from `chia-protocol`, `HashMap` from `std::collections`, `CoinRecord` from local crate.
- **Rollback modified_coins:** Contains entries for both kinds of modifications:
  - Deleted coins (created after target): the record as it was before deletion.
  - Un-spent coins (spent after target): the record with `spent_height` reverted to `None`.
- **Chia parity:** Chia's `rollback_to_block()` returns `dict[bytes32, CoinRecord]` where the CoinRecord reflects post-rollback state. dig-coinstore follows this pattern.
- **Edge cases:**
  - Rolling back to current height: no-op, empty modified_coins, counts = 0.
  - Rolling back to -1: everything deleted, all coins in modified_coins.
