## Summary

`CoinStoreConfig` contains all tuneable parameters for the coinstate store with sensible defaults. It uses a builder pattern with `with_*` methods for ergonomic construction, allowing callers to override only the parameters they care about while keeping defaults for everything else.

The config covers storage backend selection (LMDB vs RocksDB), storage path, snapshot retention, query limits, and backend-specific tuning parameters (LMDB map size, RocksDB write buffer and open files, bloom filters).

## Specification

### Struct Definition

```rust
pub struct CoinStoreConfig {
    /// Storage backend to use.
    /// Default: LMDB if `lmdb-storage` feature is enabled, else RocksDB.
    pub backend: StorageBackend,

    /// Path to the storage directory.
    pub storage_path: PathBuf,

    /// Maximum number of state snapshots to retain.
    /// Default: 10. Older snapshots are pruned automatically.
    pub max_snapshots: usize,

    /// Maximum number of results for batch queries.
    /// Default: 50,000. Matches Chia's `max_items` parameter
    /// (coin_store.py:355, coin_store.py:415).
    pub max_query_results: usize,

    /// LMDB map size (maximum database size).
    /// Default: 10 GB.
    pub lmdb_map_size: usize,

    /// RocksDB write buffer size.
    /// Default: 64 MB.
    pub rocksdb_write_buffer_size: usize,

    /// RocksDB max open files.
    /// Default: 1000.
    pub rocksdb_max_open_files: i32,

    /// Enable bloom filters for point lookups (RocksDB only).
    /// Default: true. 10 bits per key (~1% false positive rate).
    pub bloom_filter: bool,
}

pub enum StorageBackend {
    /// LMDB -- optimized for read-heavy, fast point lookups.
    Lmdb,
    /// RocksDB -- optimized for write-heavy workloads with bloom filters.
    RocksDb,
}
```

## Implementation Notes

- **Primary file:** `src/config.rs`
- **Builder pattern:** Each `with_*` method takes `self` by value and returns `Self`, enabling: `CoinStoreConfig::default().with_backend(StorageBackend::RocksDb).with_bloom_filter(true)`.
- **Feature gates:** The default `backend` is conditional on feature flags: `lmdb-storage` -> `Lmdb`, `rocksdb-storage` -> `RocksDb`. If both enabled, LMDB is preferred.
- **PathBuf:** `with_storage_path()` accepts `impl Into<PathBuf>` for ergonomic `&str` / `String` / `PathBuf` usage.
- **Constants:** Default values should reference the named constants from SPEC Section 2.7 (e.g., `DEFAULT_MAX_SNAPSHOTS`, `DEFAULT_MAX_QUERY_RESULTS`, `DEFAULT_LMDB_MAP_SIZE`).
