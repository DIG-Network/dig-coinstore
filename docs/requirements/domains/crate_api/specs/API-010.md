# API-010: RollbackAboveTip Error Variant and is_unspent Method

> **Authoritative requirement:** [API-010](../NORMATIVE.md#API-010)

> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

The `CoinStoreError` enum MUST include a `RollbackAboveTip` variant that is returned when a rollback is requested to a height above the current chain tip. This provides a clear, typed error for an invalid rollback target rather than relying on generic error messages.

Additionally, `CoinStore` MUST expose a public `is_unspent()` method that performs an O(1) lock-free lookup against the in-memory unspent set (PRF-001). This method is critical for fast removal validation and external callers that need to check coin existence without acquiring a full read lock.

---

## Specification

### RollbackAboveTip Variant

```rust
#[derive(Debug, Clone, PartialEq, thiserror::Error)]
pub enum CoinStoreError {
    // ... existing variants ...

    #[error("cannot rollback: target height {target} above current height {current}")]
    RollbackAboveTip { target: i64, current: u64 },
}
```

The `target` field is `i64` to allow representing negative heights or other invalid values that a caller might pass. The `current` field is `u64` matching the standard height type.

### is_unspent Method

```rust
impl CoinStore {
    /// Check if a coin is currently unspent.
    /// O(1) lock-free lookup against the in-memory HashSet<CoinId>.
    pub fn is_unspent(&self, coin_id: &CoinId) -> bool;
}
```

### Semantics

- `is_unspent()` reads from the in-memory `HashSet<CoinId>` (PRF-001).
- The method MUST NOT acquire a write lock or any mutex.
- The method returns `true` if the coin is in the unspent set, `false` otherwise.
- A `false` return does not distinguish between "coin exists but is spent" and "coin does not exist."

### RollbackAboveTip Trigger

`rollback_to_block(target_height)` MUST return `Err(CoinStoreError::RollbackAboveTip { target, current })` when `target_height > current_height`.

## Acceptance Criteria

- [x] `CoinStoreError::RollbackAboveTip` variant exists with fields `target: i64` and `current: u64`.
- [x] `rollback_to_block()` returns `RollbackAboveTip` when target > current height.
- [x] `CoinStore::is_unspent()` is publicly exported.
- [x] `is_unspent()` returns `bool`.
- [x] `is_unspent()` performs O(1) lookup (no storage I/O).
- [x] `is_unspent()` is lock-free (no write lock or mutex acquisition).
- [x] `is_unspent()` returns `true` for unspent coins, `false` for spent or nonexistent coins.

---

## Implementation Notes

- **Primary file:** `src/error.rs` (for RollbackAboveTip), `src/coin_store.rs` (for is_unspent)
- **Dependencies:** PRF-001 (in-memory unspent set), rollback logic
- **Edge cases:**
  - `is_unspent()` called with a coin_id that was never created: returns `false`.
  - `is_unspent()` called during `apply_block` Phase 2: the unspent set is being mutated under an exclusive lock, so concurrent `is_unspent()` calls would block on the read lock. This is acceptable; "lock-free" refers to the internal implementation not using additional synchronization beyond the RwLock.
  - `RollbackAboveTip` with `target == current`: this is a no-op rollback, not an error. The error only triggers when `target > current`.
