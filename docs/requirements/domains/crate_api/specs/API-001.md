## Summary

The `CoinStore` struct provides two constructors: `new()` for creating a coinstate store with sensible defaults at a given path, and `with_config()` for customizing all parameters via `CoinStoreConfig`. Additionally, `init_genesis()` bootstraps the chain with initial coins.

This mirrors Chia's `CoinStore` initialization pattern, where the store is constructed and then populated with genesis state. Chia does not have a separate `init_genesis()` method -- genesis coins are applied via the first `new_block()` call ([`coin_store.py:105-178`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/coin_store.py#L105)). dig-coinstore separates genesis initialization to make the bootstrap step explicit and to prevent double-initialization.

All constructors return `Result<Self, CoinStoreError>` because storage initialization may fail (permissions, corrupt data, backend unavailable).

## Specification

### Function Signatures

```rust
impl CoinStore {
    /// Create a new coinstate store with default configuration.
    pub fn new(path: impl AsRef<Path>) -> Result<Self, CoinStoreError>;

    /// Create a coinstate store with custom configuration.
    pub fn with_config(config: CoinStoreConfig) -> Result<Self, CoinStoreError>;

    /// Initialize genesis state with initial coins.
    /// Called once when bootstrapping a new chain.
    pub fn init_genesis(
        &mut self,
        initial_coins: Vec<(Coin, bool)>,  // (coin, is_coinbase)
        timestamp: u64,
    ) -> Result<Bytes32, CoinStoreError>;  // returns genesis state root
}
```

### Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `path` | `impl AsRef<Path>` | Filesystem path for the storage directory. |
| `config` | `CoinStoreConfig` | Custom configuration. See [API-003](API-003.md) for all fields and defaults. |
| `initial_coins` | `Vec<(Coin, bool)>` | Genesis coins with coinbase flag. |
| `timestamp` | `u64` | Genesis block timestamp (unix seconds). |

### new() Behavior

1. Construct `CoinStoreConfig` with `storage_path` set to the provided path and all other fields at defaults.
2. Open or create the storage backend (LMDB or RocksDB based on feature flags).
3. Initialize internal data structures (empty Merkle tree, empty indices, zero counters).
4. Return `Ok(CoinStore)` or `Err(CoinStoreError::StorageError)` on failure.

### with_config() Behavior

1. Use the provided `config` as-is (all fields including `storage_path`).
2. Open or create the storage backend per `config.backend`.
3. Initialize internal data structures.
4. Return `Ok(CoinStore)` or `Err(CoinStoreError::StorageError)` on failure.

### init_genesis() Behavior

1. Verify the coinstate has not been previously initialized. If `height() > 0` or coins exist, return `Err(CoinStoreError::GenesisAlreadyInitialized)`.
2. Insert all `initial_coins` as coin records at height 0 with the provided `timestamp`.
3. Build the initial Merkle tree over all genesis coins.
4. Set chain tip to height 0 with a zero-hash block hash.
5. Return `Ok(state_root)` with the computed genesis state root.

## Acceptance Criteria

- [ ] `CoinStore::new()` is publicly exported and accepts a path.
- [ ] `CoinStore::with_config()` is publicly exported and accepts `CoinStoreConfig`.
- [ ] Both constructors return `Result<Self, CoinStoreError>`.
- [ ] `init_genesis()` is publicly exported and returns `Result<Bytes32, CoinStoreError>`.
- [ ] `new()` produces a coinstate store with default configuration.
- [ ] `with_config()` respects all provided configuration values.
- [ ] `init_genesis()` fails with `GenesisAlreadyInitialized` if called twice.
- [ ] After `init_genesis()`, `height()` returns 0 and `state_root()` returns the genesis root.
- [ ] `is_empty()` returns true before `init_genesis()`.

## Implementation Notes

- **Primary file:** `src/store.rs` or `src/coinstore.rs`
- **Storage initialization:** The storage backend (LMDB via `heed`, RocksDB via `rocksdb`) is opened at the given path. Directories are created if they do not exist.
- **Feature gates:** `lmdb-storage` enables LMDB backend, `rocksdb-storage` enables RocksDB. At least one must be enabled.
- **Edge cases:**
  - `new()` on a path with existing data: re-opens the existing store (not a fresh init).
  - `init_genesis()` with empty `initial_coins`: valid (empty genesis).
  - `with_config()` with nonexistent `storage_path`: created automatically.
