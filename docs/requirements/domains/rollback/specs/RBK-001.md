# RBK-001: rollback_to_block() Entry Point

> **Authoritative requirement:** [RBK-001](../NORMATIVE.md#RBK-001)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

---

## Summary

`rollback_to_block()` is the primary entry point for chain reorganization in the CoinStore. It reverses coin state changes back to a specified target height, enabling the node to handle forks by undoing blocks that are no longer on the canonical chain. The function accepts a signed integer target height, where negative values indicate a full reset (roll back all blocks).

This mirrors Chia L1's `rollback()` method in `coin_store.py` which iterates over coins confirmed or spent after the target height and reverses their state changes.

---

## Specification

### Function Signature

```rust
impl CoinStore {
    pub fn rollback_to_block(&mut self, target_height: i64) -> Result<RollbackResult, CoinStoreError>;
}
```

### Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `target_height` | `i64` | The block height to roll back to (inclusive). Coins confirmed at or before this height are preserved. Negative values trigger a full reset. |

### Return Type

- **`Ok(RollbackResult)`** -- Contains `modified_coins: HashMap<CoinId, CoinRecord>` with pre-modification snapshots of all affected coins, plus the new chain height.
- **`Err(CoinStoreError)`** -- Storage or consistency error during rollback.

### Behavior

1. If `target_height < 0`, treat as rolling back all blocks (height 0, empty state).
2. Delete all coins confirmed after `target_height` (see RBK-002).
3. Un-spend all coins spent after `target_height` (see RBK-003).
4. Recompute FF eligibility for un-spent coins (see RBK-004).
5. Update chain tip height to `target_height` (clamped to 0 minimum).
6. Return `RollbackResult` with all modified coin records.

## Acceptance Criteria

- [ ] The method `rollback_to_block()` exists on `CoinStore` with the exact signature specified above.
- [ ] `target_height` of `-1` rolls back all blocks, leaving the store empty.
- [ ] `target_height` equal to the current height is a no-op returning an empty `modified_coins`.
- [ ] `target_height` greater than the current height is a no-op.
- [ ] Returns `RollbackResult` with pre-modification snapshots of all affected coins.
- [ ] Chain height is updated to `max(target_height, 0)` after rollback.

---

## Implementation Notes

- **Primary file:** `src/rollback.rs`
- **Dependencies:** RBK-002 (deletion), RBK-003 (un-spending), RBK-004 (FF recomputation)
- **Edge cases:**
  - Negative `target_height` must not panic; clamp to 0 after full reset.
  - Rolling back to height 0 should preserve genesis coins (confirmed at height 0).
  - Empty store rollback is a no-op.
