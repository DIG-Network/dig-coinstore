# RBK-003: Coin Un-spending During Rollback

> **Authoritative requirement:** [RBK-003](../NORMATIVE.md#RBK-003)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

---

## Summary

During rollback, coins that were spent after the target height but were confirmed at or before the target height must be marked as unspent. This restores them to their pre-spend state. These coins are added to `modified_coins` only if they were not already captured by the deletion pass (RBK-002), preventing duplicate entries.

This corresponds to the second phase of Chia L1's `rollback()` method, where coins with `spent_index > fork_height` are updated to have `spent_index = 0`.

---

## Specification

### Behavior

1. Query all coins where `spent_height > target_height` AND `confirmed_height <= target_height`.
2. For each such coin:
   a. Save the pre-modification `CoinRecord` to `modified_coins` if not already present (dedup with RBK-002).
   b. Set `spent_height = None` (unspent).
   c. Recompute `ff_eligible` (see RBK-004).
   d. Re-insert into the `unspent_by_puzzle_hash` index.
   e. Remove from the `coin_by_spent_height` index.
   f. Update the in-memory unspent set (PRF-001).
3. Update materialized counters (increment unspent_count, decrement spent_count).

### Deduplication with Deletion Pass

A coin created at height 3 and spent at height 7, when rolling back to height 2, is handled entirely by RBK-002 (deletion). The un-spending pass (this requirement) only handles coins that remain in the store (confirmed at or before target height).

---

## Acceptance Criteria

- [ ] All coins with `spent_height > target_height` and `confirmed_height <= target_height` are marked unspent.
- [ ] `spent_height` is set to `None` for un-spent coins.
- [ ] Pre-modification snapshots are added to `modified_coins` only if not already present from deletion pass.
- [ ] `unspent_by_puzzle_hash` index is updated (coin re-inserted).
- [ ] `coin_by_spent_height` index is cleaned up (entry removed).
- [ ] In-memory unspent set is updated.
- [ ] Materialized counters are updated correctly.

---

## Implementation Notes

- **Primary file:** `src/rollback.rs`
- **Dependencies:** RBK-002 (runs first), RBK-004 (FF recomputation), PRF-001 (unspent set), PRF-003 (counters), PRF-004 (unspent index)
- **Edge cases:**
  - Coin spent at exactly `target_height` is NOT un-spent (only coins spent AFTER target).
  - Multiple coins spent in the same block: all handled together.
