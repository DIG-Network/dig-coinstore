# RBK-005: rollback_n_blocks() Convenience Wrapper

> **Authoritative requirement:** [RBK-005](../NORMATIVE.md#RBK-005)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

---

## Summary

`rollback_n_blocks()` is a convenience method that rolls back the most recent `n` blocks by computing the target height from the current chain height and delegating to `rollback_to_block()`. This simplifies the common case of undoing a fixed number of recent blocks.

---

## Specification

### Function Signature

```rust
impl CoinStore {
    pub fn rollback_n_blocks(&mut self, n: u64) -> Result<RollbackResult, CoinStoreError>;
}
```

### Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `n` | `u64` | The number of blocks to roll back from the current height. |

### Behavior

1. Compute `target_height = self.height() - n` (as `i64` to allow underflow to negative).
2. Delegate to `self.rollback_to_block(target_height)`.
3. Return the result unchanged.

### Edge Cases

- `n = 0`: rolls back to current height, which is a no-op.
- `n > self.height()`: results in a negative `target_height`, triggering a full reset (per RBK-001).

---

## Acceptance Criteria

- [ ] The method `rollback_n_blocks()` exists on `CoinStore` with the specified signature.
- [ ] `rollback_n_blocks(1)` rolls back exactly one block.
- [ ] `rollback_n_blocks(0)` is a no-op.
- [ ] `rollback_n_blocks(n)` where `n > height` triggers a full reset.
- [ ] The result is identical to calling `rollback_to_block(height - n)` directly.

## Implementation Notes

- **Primary file:** `src/rollback.rs`
- **Dependencies:** RBK-001 (delegates to rollback_to_block)
