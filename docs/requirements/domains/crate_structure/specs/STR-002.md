## Summary

The module hierarchy defines the physical file layout and logical module structure of the `dig-coinstore` crate. `src/lib.rs` serves as the crate root, declaring all modules and re-exporting the public API surface. Each functional concern -- types, errors, configuration, block application, rollback, queries, hints, storage, merkle trees, caching, and archival -- lives in its own dedicated file or subdirectory module.

This layout is derived from the working code in `l2_driver_state_channel` and organized to support separation of concerns, independent testing, and clear ownership boundaries. Subdirectory modules (`storage/`, `merkle/`, `cache/`) group related files under a `mod.rs` root.

## Specification

### Crate Root

`src/lib.rs` MUST:
1. Declare all top-level modules via `mod` statements.
2. Re-export all public types, structs, enums, and traits that form the crate's public API.
3. Use `#[cfg(feature = "...")]` guards on storage backend module declarations.

### Top-Level Source Files

| File | Responsibility |
|------|---------------|
| `src/lib.rs` | Crate root, module declarations, re-exports |
| `src/coin_store.rs` | `CoinStore` struct, primary public API orchestration |
| `src/config.rs` | `CoinStoreConfig` struct and configuration constants |
| `src/error.rs` | `CoinStoreError` enum with `thiserror` derivation |
| `src/types.rs` | `BlockData`, `CoinAddition`, `CoinRecord`, `CoinState`, and other domain types |
| `src/block_apply.rs` | Block application pipeline (Phase 1: validation, Phase 2: mutation) |
| `src/rollback.rs` | Rollback pipeline for chain reorganization |
| `src/queries.rs` | All query method implementations on `CoinStore` |
| `src/hints.rs` | Hint store logic for puzzle hash hints |
| `src/archive.rs` | Tiered spent coin archival |

### Subdirectory Modules

| Directory | `mod.rs` Responsibility | Child Files |
|-----------|------------------------|-------------|
| `src/storage/` | `StorageBackend` trait definition | `rocksdb.rs`, `lmdb.rs`, `schema.rs` |
| `src/merkle/` | `SparseMerkleTree` struct | `proof.rs`, `persistent.rs` |
| `src/cache/` | Cache module root, re-exports | `unspent_set.rs`, `lru_cache.rs`, `counters.rs` |

## Implementation Notes

- **Primary file:** `src/lib.rs`
- **Dependencies:** All other STR requirements (STR-001 for deps, STR-003 through STR-006 for module contents)
- **Edge cases:**
  - Storage backend modules (`rocksdb.rs`, `lmdb.rs`) MUST be conditionally compiled with `#[cfg(feature = "...")]` in the parent `mod.rs`.
  - Test files are integration tests (under `tests/`), not unit tests (under `src/`). Each test file is compiled as a separate crate by Cargo.
