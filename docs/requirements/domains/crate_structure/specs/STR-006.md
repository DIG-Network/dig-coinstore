## Summary

The test infrastructure provides a consistent, reusable foundation for all integration tests in `dig-coinstore`. The `tests/` directory contains one test file per requirement domain (e.g., `blk_tests.rs` for block application, `qry_tests.rs` for queries), and a shared `tests/helpers/mod.rs` module that provides common utilities used across all test files.

This structure ensures that tests are organized by functional concern, that shared setup logic is not duplicated, and that each test domain can be run independently. The helpers module provides coin builder functions, temporary directory management for storage backends, genesis state initialization, and block builder helpers.

## Specification

### Test File Organization

Each requirement domain MUST have a corresponding test file in `tests/`:

| Domain | Test File | Prefix |
|--------|-----------|--------|
| Crate Structure | `tests/str_tests.rs` | `STR` |
| Crate API | `tests/api_tests.rs` | `API` |
| Block Application | `tests/blk_tests.rs` | `BLK` |
| Rollback | `tests/rbk_tests.rs` | `RBK` |
| Queries | `tests/qry_tests.rs` | `QRY` |
| Storage | `tests/sto_tests.rs` | `STO` |
| Merkle | `tests/mrk_tests.rs` | `MRK` |
| Hints | `tests/hnt_tests.rs` | `HNT` |
| Performance | `tests/prf_tests.rs` | `PRF` |
| Concurrency | `tests/con_tests.rs` | `CON` |

### Helpers Module

`tests/helpers/mod.rs` MUST provide the following shared utilities:

| Utility | Description |
|---------|-------------|
| Coin builder functions | Create `Coin`, `CoinRecord`, and `CoinState` instances with sensible defaults and builder-pattern overrides |
| Temporary directory management | Create and clean up temp directories for storage backend tests (using `tempfile` or equivalent) |
| Genesis state initialization | Initialize a `CoinStore` with a standard genesis state for repeatable tests |
| Block builder helpers | Construct `BlockData` instances with configurable additions, removals, and hints |

### Import Convention

All test files MUST import the helpers module using:

```rust
mod helpers;
```

This leverages Cargo's integration test module resolution, where `tests/helpers/mod.rs` is recognized as a shared module (not a standalone test file).

### Test Naming Convention

Test functions SHOULD follow the pattern:

```
test_<domain_prefix>_<requirement_id>_<description>
```

For example: `test_blk_001_apply_single_block`, `test_qry_003_by_puzzle_hash`.

## Acceptance Criteria

- [ ] `tests/helpers/mod.rs` exists and is non-empty.
- [ ] Helpers module provides coin builder functions.
- [ ] Helpers module provides temporary directory management.
- [ ] Helpers module provides genesis state initialization.
- [ ] Helpers module provides block builder helpers.
- [ ] All ten domain test files exist under `tests/`.
- [ ] Each test file contains `mod helpers;` as an import.
- [ ] Helpers module compiles when imported from any test file.
- [ ] `cargo test` discovers and runs tests from all test files.
- [ ] Test files are independent -- each can be run individually via `cargo test --test <name>`.

## Implementation Notes

- **Primary files:** `tests/helpers/mod.rs`, `tests/*.rs`
- **Dependencies:** STR-001 (dev-dependencies), STR-002 (test directory layout)
- **Edge cases:**
  - The `helpers/` directory MUST contain a `mod.rs` (not just a `helpers.rs` file) to be recognized as a module by Cargo's integration test resolver.
  - Performance tests (`prf_tests.rs`) may need longer timeouts or `#[ignore]` attributes for CI.
  - Concurrency tests (`con_tests.rs`) should use thread counts appropriate for CI environments.
  - The `tempfile` crate should be declared as a `[dev-dependencies]` in `Cargo.toml` for temporary directory management.

## Verification

### Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| `test_helpers_compile` | Build | Import helpers module, call a builder function | Compiles and runs |
| `test_coin_builder` | Unit | Use coin builder to create a Coin with custom fields | Coin fields match |
| `test_temp_dir_cleanup` | Unit | Create temp dir, verify it exists, drop handle, verify cleanup | Dir cleaned up |
| `test_genesis_init` | Integration | Initialize CoinStore with genesis state, verify height 0 | Height is 0, genesis coins present |
| `test_block_builder` | Unit | Build a BlockData with additions and removals | BlockData fields match |
| `test_all_test_files_exist` | Static | Verify all expected test files are present | All files exist |
| `test_helpers_imported` | Static | Grep each test file for `mod helpers;` | All test files import helpers |
| `test_independent_execution` | Build | Run each test file independently via `cargo test --test <name>` | Each passes independently |
