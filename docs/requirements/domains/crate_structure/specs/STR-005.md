# STR-005: Re-export Strategy

## Summary

The re-export strategy ensures that `dig-coinstore` uses Chia ecosystem types (`Coin`, `Bytes32`, `CoinState`) exclusively through `dig-clvm` re-exports rather than defining its own duplicate types. This maintains a single integration point for all Chia protocol types across the DIG crate ecosystem, preventing type mismatches, redundant definitions, and version skew between crates.

`src/lib.rs` MUST re-export these types so that consumers of `dig-coinstore` can access them without adding a direct dependency on `chia-protocol` or `dig-clvm`. The crate MUST NOT define its own `Coin`, `Bytes32`, or `CoinState` structs.

---

## Specification

### Re-exports in lib.rs

`src/lib.rs` MUST contain re-export statements for the following types from `dig-clvm`:

```rust
// Re-export Chia types via dig-clvm (single integration point)
pub use dig_clvm::Coin;
pub use dig_clvm::Bytes32;
pub use dig_clvm::CoinState;
```

### Prohibition on Redefinition

The crate MUST NOT contain any of the following:
- A `struct Coin` definition (in any module).
- A `struct Bytes32` definition (in any module).
- A `struct CoinState` definition (in any module).
- Type aliases that shadow these names with different underlying types.

### Dependency Chain

The type flow MUST follow this chain:

```
chia-protocol  -->  dig-clvm  -->  dig-coinstore  -->  consumers
    (defines)      (re-exports)    (re-exports)
```

This ensures all crates in the DIG ecosystem reference the same concrete types, enabling seamless interop between `dig-coinstore`, `dig-mempool`, and other DIG crates.

### Additional Types

Beyond the three mandatory re-exports, `src/lib.rs` SHOULD also re-export any other `chia-protocol` types that appear in the public API of `dig-coinstore` (e.g., `Program`, `Signature`), always via `dig-clvm`.

---

## Acceptance Criteria

- [ ] `src/lib.rs` contains `pub use dig_clvm::Coin;`.
- [ ] `src/lib.rs` contains `pub use dig_clvm::Bytes32;`.
- [ ] `src/lib.rs` contains `pub use dig_clvm::CoinState;`.
- [ ] No file in `src/` defines `struct Coin`.
- [ ] No file in `src/` defines `struct Bytes32`.
- [ ] No file in `src/` defines `struct CoinState`.
- [ ] `dig_coinstate::Coin` and `dig_clvm::Coin` are the same type (compile-time check).
- [ ] `dig_coinstate::Bytes32` and `dig_clvm::Bytes32` are the same type (compile-time check).
- [ ] Consumers can use `dig_coinstate::Coin` without depending on `dig-clvm` directly.

---

## Implementation Notes

- **Primary file:** `src/lib.rs`
- **Dependencies:** STR-001 (dig-clvm dependency in Cargo.toml), STR-002 (lib.rs as crate root)
- **Edge cases:**
  - If `dig-clvm` changes the re-export path for Chia types, `dig-coinstore` must update accordingly.
  - Version pinning of `dig-clvm` ensures type compatibility. If `dig-clvm` bumps its `chia-protocol` dependency, `dig-coinstore` must follow.
  - The re-exports MUST be at the crate root level (not nested in a submodule) for ergonomic access.
