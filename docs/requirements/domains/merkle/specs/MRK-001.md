# MRK-001: SparseMerkleTree

> **Authoritative requirement:** [MRK-001](../NORMATIVE.md#MRK-001)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

`SparseMerkleTree` is the core data structure for computing a cryptographic commitment over the entire coin set. It is a 256-level binary trie keyed by `Bytes32` coin IDs, where each leaf holds the hash of the coin record and each internal node holds the hash of its two children. The tree supports batch mutation operations (`batch_insert`, `batch_update`, `batch_remove`) that defer root hash recomputation until `root()` is explicitly called, ensuring at most one expensive tree traversal per block application.

This design follows the sparse Merkle tree construction described in SPEC.md Section 9, where the depth is fixed at 256 to match the bit-length of coin IDs (SHA-256 hashes).

## Specification

### Struct

```rust
pub struct SparseMerkleTree {
    /// Non-empty leaves only: coin_id → pre-hashed leaf digest.
    leaves: HashMap<Bytes32, Bytes32>,
    /// Cached root; `None` after any successful mutation batch until `root()` runs.
    root_hash: Option<Bytes32>,
}
```

### Methods

```rust
impl SparseMerkleTree {
    pub fn batch_insert(&mut self, entries: &[(Bytes32, Bytes32)]) -> Result<(), MerkleError>;
    pub fn batch_update(&mut self, entries: &[(Bytes32, Bytes32)]) -> Result<(), MerkleError>;
    pub fn batch_remove(&mut self, keys: &[Bytes32]) -> Result<(), MerkleError>;

    /// Recomputes from `leaves` when `root_hash` is `None`, then caches the digest.
    /// Errors are surfaced only from `batch_*`; `root()` itself does not fail.
    pub fn root(&mut self) -> Bytes32;

    /// Same mathematical root as `root()` on a shared `&self` without updating the cache.
    pub fn root_observed(&self) -> Bytes32;

    /// `true` when mutations invalidated the cache but leaves remain (needs `root()` to refresh cache).
    pub fn is_dirty(&self) -> bool;
}
```

### Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `entries` | `&[(Bytes32, Bytes32)]` | Key-value pairs where key is coin_id and value is leaf hash |
| `keys` | `&[Bytes32]` | Coin IDs to remove from the tree |

### Behavior

1. `batch_insert` adds new leaves at positions determined by the 256-bit key. Each bit of the key selects left (0) or right (1) at successive tree levels. Inserting a key that already exists MUST return an error.
2. `batch_update` replaces the leaf hash for existing keys. Updating a key that does not exist MUST return an error.
3. `batch_remove` sets leaves back to the empty hash (logically removing them). Removing a key that does not exist MUST return an error.
4. All three mutation methods mark the tree as dirty but do NOT recompute the root hash.
5. `root()` recomputes the root hash from the in-memory leaf map when the cache is invalidated, then stores the digest in `root_hash` for O(1) subsequent reads until the next mutation.
6. Multiple calls to `root()` without intervening mutations MUST return the same hash without recomputation.
7. An empty tree (no leaves) MUST return the full-tree empty root `empty_hash(SMT_HEIGHT)` from the MRK-002 memoized table (same value as `SparseMerkleTree::new` starts with).

## Acceptance Criteria

- [x] `SparseMerkleTree` struct exists with `batch_insert`, `batch_update`, `batch_remove`, and `root()` methods.
- [x] `batch_insert` with N entries followed by a single `root()` call produces the correct Merkle root.
- [x] `batch_update` changes leaf values and a subsequent `root()` reflects the new state.
- [x] `batch_remove` logically removes leaves and a subsequent `root()` reflects the removal.
- [x] Mutation methods do NOT trigger root recomputation (deferred).
- [x] `root()` recomputes at most once per dirty interval.
- [x] Tree depth is exactly 256 levels.
- [x] Inserting a duplicate key returns an error.
- [x] Updating or removing a non-existent key returns an error.

## Implementation Notes

- **Primary file:** `src/merkle/mod.rs` (`SparseMerkleTree`, `MerkleError`, `merkle_leaf_hash`, `merkle_node_hash`, `empty_hash` wiring)
- **V&V:** `tests/mrk_001_tests.rs` (one integration crate per MRK-001; maps to the test plan table below)
- **Dependencies:** MRK-002 (empty hashes), MRK-003 (persistent nodes)
- **Edge cases:**
  - Empty tree returns the precomputed empty root hash.
  - Inserting and then removing the same key in a single batch should result in no net change.
  - Large batches (thousands of insertions per block) should be efficient via shared-path deduplication.
- **Performance:** Deferred recomputation means that a block inserting 1,000 coins only traverses dirty paths once, not 1,000 times.

## Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| `test_batch_insert_single` | Unit | Insert one leaf, call root() | Root matches manual 256-level hash computation |
| `test_batch_insert_multiple` | Unit | Insert 100 leaves, call root() once | Single recomputation, correct root |
| `test_batch_update` | Unit | Insert leaf, update its value, call root() | Root reflects new leaf value |
| `test_batch_remove` | Unit | Insert leaf, remove it, call root() | Root equals empty tree root |
| `test_deferred_recomputation` | Unit | Insert leaves without calling root(), verify no recompute | Dirty flag set, root_hash unchanged |
| `test_idempotent_root` | Unit | Call root() twice with no mutations | Same hash returned, no recomputation on second call |
| `test_duplicate_insert_error` | Unit | Insert same key twice | Returns MerkleError |
| `test_remove_missing_error` | Unit | Remove key not in tree | Returns MerkleError |
| `test_256_level_depth` | Unit | Insert key differing only in last bit from another | Both leaves present at correct depth-256 positions |
| `test_empty_tree_root_standalone` | Unit | Call root() on a freshly constructed tree with no mutations | Returns the precomputed empty tree root from MRK-002 |
| `test_insert_then_remove_equals_empty` | Unit | Insert one leaf, remove it, call root() | Root equals empty tree root |
| `test_batch_update_missing_key_error` | Unit | Call batch_update with a key not in the tree | Returns MerkleError |
| `test_idempotent_update` | Unit | Update a leaf with the same value it already has, call root() | Root unchanged from before the update |
