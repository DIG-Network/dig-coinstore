# MRK-002: Memoized Empty Hashes

> **Authoritative requirement:** [MRK-002](../NORMATIVE.md#MRK-002)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

---

## Summary

A sparse Merkle tree with 256 levels requires frequent access to "empty subtree" hashes: the digest of a completely empty subtree at each distance from the leaves. Naively recomputing these on every access is redundant because they are deterministic constants. This requirement mandates a pre-computed table of **257** values (indices `0..=256`), stored in a `OnceLock<[Bytes32; 257]>` for thread-safe lazy initialization and O(1) lookup.

**Indexing convention (matches `src/merkle/mod.rs` and MRK-001):**

- **`empty_hash(0)`** — domain-separated hash of the empty leaf sentinel (bottom of the tree).
- **`empty_hash(n)`** for `1 <= n <= 256` — hash of a node whose both children are the empty subtree at level `n - 1`.
- **`empty_hash(SMT_HEIGHT)`** where `SMT_HEIGHT = 256` — the root hash of an entirely empty 256-level tree (what `SparseMerkleTree::new` exposes as `root()`).

There are 257 entries because we store the leaf empty hash plus one value per level of internal hashing up to the root.

## Specification

### Data Structure

```rust
use std::sync::OnceLock;

static EMPTY_HASHES: OnceLock<[Bytes32; 257]> = OnceLock::new();

/// Returns the pre-computed empty hash at the given level (0 = leaf, SMT_HEIGHT = root).
pub fn empty_hash(level: usize) -> Bytes32;
```

### Computation

```
hashes[0] = merkle_leaf_hash(EMPTY_LEAF_SENTINEL)
hashes[i] = merkle_node_hash(hashes[i-1], hashes[i-1])   for i in 1..=256
```

Where `merkle_leaf_hash` / `merkle_node_hash` are the same domain-separated SHA-256 wrappers used by the sparse tree (MRK-001).

### Behavior

1. `EMPTY_HASHES` MUST be initialized at most once, on first access, using `OnceLock::get_or_init`.
2. The array MUST contain exactly 257 entries indexed from `0` to `256` inclusive.
3. `empty_hash(level)` MUST return `EMPTY_HASHES[level]` in O(1) time (direct array index after init).
4. `empty_hash(0)` MUST be the hash of the designated empty leaf sentinel.
5. For each `n` in `0..256`, `empty_hash(n + 1)` MUST equal `merkle_node_hash(empty_hash(n), empty_hash(n))`.
6. The implementation MUST NOT perform recursive computation at query time.

## Acceptance Criteria

- [x] `EMPTY_HASHES` is a `OnceLock<[Bytes32; 257]>` static.
- [x] Initialization computes all 257 values bottom-up in a single pass.
- [x] `empty_hash(level)` returns the correct value in O(1).
- [x] `empty_hash(0)` equals the leaf-level empty sentinel hash.
- [x] `empty_hash(SMT_HEIGHT)` equals the root hash of an entirely empty 256-level tree.
- [x] Values match those produced by naive iterative computation from the leaf.
- [x] Thread-safe: concurrent first-access from multiple threads initializes exactly once (Rust `OnceLock` semantics; exercised under contention in tests).

---

## Implementation Notes

- **Primary file:** `src/merkle/mod.rs` (empty hash table lives next to `SparseMerkleTree`; there is no separate `empty_hashes.rs` module).
- **Dependencies:** Uses the same `merkle_leaf_hash` / `merkle_node_hash` as the tree.
- **Edge cases:**
  - `empty_hash(level)` with `level > SMT_HEIGHT` MUST panic (documented); message includes `exceeds SMT_HEIGHT` for testability.
  - The empty leaf sentinel MUST stay consistent with all tree operations (see `EMPTY_LEAF_SENTINEL` in `mod.rs`).
- **Performance:** One-time initialization cost of 257 hash computations. Subsequent lookups are a single array index.
