# MRK-004: Proof Generation

> **Authoritative requirement:** [MRK-004](../NORMATIVE.md#MRK-004)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

---

## Summary

`get_coin_proof(coin_id)` generates a Merkle proof for a given coin ID against the current tree state. The proof consists of the 256 sibling hashes encountered when traversing from the root to the leaf position determined by the coin ID's bits. For existing coins, this is an inclusion proof (the leaf contains the coin's hash). For non-existing coins, this is a non-inclusion proof (the leaf position is empty, i.e., contains the empty leaf hash). Both proof types share the same structure and can be verified with the same algorithm (MRK-005).

---

## Specification

### Proof Structure

```rust
pub struct SparseMerkleProof {
    /// The coin_id (key) this proof is for.
    pub key: Bytes32,
    /// The leaf value at the key's position (coin record hash if present, empty hash if absent).
    pub leaf_value: Bytes32,
    /// Sibling hashes from leaf (index 0) to root (index 255).
    /// siblings[i] is the sibling at tree level (256 - i).
    pub siblings: [Bytes32; 256],
}
```

### Method

```rust
impl SparseMerkleTree {
    /// Generate a Merkle proof for the given coin_id.
    /// Works for both existing coins (inclusion) and non-existing coins (non-inclusion).
    pub fn get_coin_proof(&self, coin_id: &Bytes32) -> Result<SparseMerkleProof, MerkleError>;
}
```

### Behavior

1. Traverse the tree from root to leaf, following the path dictated by the 256 bits of `coin_id`. Bit 0 (most significant) selects at level 0, bit 255 (least significant) selects at level 255.
2. At each level, record the sibling node's hash. If the sibling subtree is empty, use the pre-computed empty hash for that level (MRK-002).
3. At the leaf level (256), record the leaf value. For an existing coin, this is the hash of the coin record. For a non-existing coin, this is the empty leaf hash.
4. Internal nodes not in memory MUST be fetchable from the `merkle_nodes` CF (MRK-003 lazy loading).
   **Current implementation:** when the authoritative leaf map is resident (including after
   [`SparseMerkleTree::load_from_store`]), sibling hashes are recomputed from that multiset and
   match the persisted root without per-level CF reads. A future path may hydrate siblings from disk
   when only a partial leaf set is loaded.
5. The proof MUST be valid for verification against the current root hash (MRK-005).
6. If the tree is dirty (mutations pending without `root()` call), `get_coin_proof` MUST return an error or trigger root recomputation first.

## Acceptance Criteria

- [x] `get_coin_proof` exists on `SparseMerkleTree` with the specified signature.
- [x] Returns `SparseMerkleProof` with exactly 256 sibling hashes (`Vec` wire; length enforced).
- [x] For an existing coin, `SparseMerkleProof::leaf_value()` equals the coin record hash.
- [x] For a non-existing coin, `leaf_value` equals the empty leaf hash.
- [x] Generated proof passes `verify(proof, root())` (MRK-005).
- [x] Sibling hashes for empty subtrees use pre-computed values from MRK-002.
- [x] Proof generation on a dirty tree returns an error or forces recomputation (`MerkleError::ProofRequiresCleanTree`).

## Implementation Notes

- **Primary file:** `src/merkle/proof.rs`
- **Dependencies:** MRK-001 (tree traversal), MRK-002 (empty hashes for sparse siblings), MRK-003 (lazy node loading from `merkle_nodes` CF)
- **Edge cases:**
  - Proof for a coin in a completely empty tree: all 256 siblings are empty hashes at their respective levels.
  - Proof for a coin at a path where many levels are empty subtrees: most siblings come from the pre-computed array.
  - Concurrent proof generation: since `get_coin_proof` is read-only (assuming clean tree), it should be safe for concurrent access with appropriate synchronization on the node cache.
- **Performance:** Proof generation requires at most 256 node lookups (one per level). In practice, many siblings are empty subtree hashes retrieved in O(1) from the memoized array.
