# QRY-009: Aggregate Queries

> **Authoritative requirement:** [QRY-009](../NORMATIVE.md#QRY-009)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

---

## Summary

Aggregate queries provide summary statistics about the coin store without enumerating individual records. These include counts of unspent coins, total unspent value, per-puzzle-hash aggregation, and total coin count. Implementations SHOULD use materialized counters (PRF-003) for O(1) performance rather than scanning the full dataset.

---

## Specification

### Function Signatures

```rust
use std::collections::HashMap;

impl CoinStore {
    pub fn num_unspent(&self) -> Result<u64, CoinStoreError>;
    pub fn total_unspent_value(&self) -> Result<u128, CoinStoreError>;

    /// SPEC Section 3.11 -- Full-table aggregation across all puzzle hashes.
    /// Returns a map from puzzle_hash to (total_amount, coin_count) for every
    /// puzzle hash that has at least one unspent coin.
    pub fn aggregate_unspent_by_puzzle_hash(
        &self,
    ) -> Result<HashMap<Bytes32, (u64, usize)>, CoinStoreError>;

    pub fn num_total(&self) -> Result<u64, CoinStoreError>;
}
```

### Behavior

- `num_unspent()`: Returns the count of all unspent coins. SHOULD read from materialized counter.
- `total_unspent_value()`: Returns the sum of amounts of all unspent coins. SHOULD read from materialized counter.
- `aggregate_unspent_by_puzzle_hash()` (SPEC Section 3.11): Performs a full-table scan of unspent coins, grouping by puzzle hash. Returns a `HashMap<Bytes32, (u64, usize)>` where each entry maps a puzzle hash to `(total_amount, coin_count)`. Only puzzle hashes with at least one unspent coin appear in the map.
- `num_total()`: Returns the total number of coins (spent + unspent). SHOULD read from materialized counter.

---

## Acceptance Criteria

- [ ] `num_unspent()` returns correct count of unspent coins.
- [ ] `total_unspent_value()` returns correct sum of unspent coin amounts.
- [ ] `aggregate_unspent_by_puzzle_hash()` returns a `HashMap<Bytes32, (u64, usize)>` covering all puzzle hashes with unspent coins.
- [ ] Each map entry contains the correct `(total_amount, coin_count)` for its puzzle hash.
- [ ] Puzzle hashes with no unspent coins are absent from the map.
- [ ] `num_total()` returns correct total coin count.
- [ ] All aggregate queries are consistent with the current store state.
- [ ] Performance is O(1) for counter-backed queries (`num_unspent`, `total_unspent_value`, `num_total`).

---

## Implementation Notes

- **Primary file:** `src/queries.rs`
- **Dependencies:** PRF-003 (materialized counters), PRF-004 (unspent index)
- **Edge cases:**
  - Empty store: all counts 0, all values 0; `aggregate_unspent_by_puzzle_hash()` returns empty HashMap.
  - After rollback: counters must reflect rolled-back state.
  - `total_unspent_value` uses u128 to avoid overflow with large coin amounts.
  - `aggregate_unspent_by_puzzle_hash()` performs a full-table scan; callers should be aware of cost on large stores.
