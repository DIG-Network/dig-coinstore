# QRY-005: Name Queries with Filtering

> **Authoritative requirement:** [QRY-005](../NORMATIVE.md#QRY-005)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

---

## Summary

Name queries retrieve coin records by their coin IDs (names) with additional filtering by spent status and height range. Unlike the basic point lookups in QRY-001, these queries support `include_spent` and height range parameters, making them suitable for filtered bulk retrieval.

This mirrors Chia L1's `get_coin_records_by_names()` method.

---

## Specification

### Function Signature

```rust
impl CoinStore {
    pub fn get_coin_records_by_names(
        &self,
        names: &[Bytes32],
        include_spent: bool,
        start_height: Option<u32>,
        end_height: Option<u32>,
    ) -> Result<Vec<CoinRecord>, CoinStoreError>;
}
```

### Behavior

- Looks up coins by coin ID (same as QRY-001 batch), then applies post-filters.
- If `include_spent` is `false`, spent coins are excluded from results.
- `start_height` and `end_height` filter by `confirmed_height` (inclusive).
- Missing coin IDs are silently skipped.

---

## Acceptance Criteria

- [ ] Returns records for all found coin IDs matching filters.
- [ ] `include_spent=false` excludes spent coins.
- [ ] Height range filtering works correctly.
- [ ] Missing coin IDs are silently skipped.
- [ ] Empty names slice returns empty result.

---

## Implementation Notes

- **Primary file:** `src/queries.rs`
- **Dependencies:** QRY-001 (underlying point lookup)
- **Edge cases:**
  - All requested names filtered out: returns empty vector.
  - Duplicate names in input: return one record per unique name.
