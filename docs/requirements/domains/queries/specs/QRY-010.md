# QRY-010: Chain State Metadata

> **Authoritative requirement:** [QRY-010](../NORMATIVE.md#QRY-010)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

---

## Summary

Chain state metadata queries provide access to the current state of the coin store's chain tracking. These include the current block height, tip block hash, Merkle state root, timestamp, aggregate statistics, and an emptiness check. These are essential for sync protocols, health checks, and API responses.

---

## Specification

### Function Signatures

```rust
impl CoinStore {
    pub fn height(&self) -> u32;
    pub fn tip_hash(&self) -> Bytes32;
    pub fn state_root(&self) -> Bytes32;
    pub fn timestamp(&self) -> u64;
    pub fn stats(&self) -> CoinStoreStats;
    pub fn is_empty(&self) -> bool;
}
```

### CoinStoreStats Struct

```rust
pub struct CoinStoreStats {
    pub height: u32,
    pub num_unspent: u64,
    pub num_spent: u64,
    pub num_total: u64,
    pub total_unspent_value: u128,
}
```

### Behavior

- `height()`: Returns the current chain tip height. 0 for empty/genesis store.
- `tip_hash()`: Returns the block hash of the current tip block.
- `state_root()`: Returns the Merkle root of the sparse Merkle tree (MRK-001).
- `timestamp()`: Returns the timestamp of the current tip block.
- `stats()`: Returns aggregate statistics in a single struct.
- `is_empty()`: Returns `true` if no blocks have been applied.

---

## Acceptance Criteria

- [ ] `height()` returns the correct current height after block application and rollback.
- [ ] `tip_hash()` returns the hash of the most recently applied block.
- [ ] `state_root()` returns the current Merkle root.
- [ ] `timestamp()` returns the timestamp of the current tip.
- [ ] `stats()` returns consistent aggregate statistics.
- [ ] `is_empty()` returns `true` for a fresh store, `false` after applying a block.

---

## Implementation Notes

- **Primary file:** `src/queries.rs` or `src/coin_store.rs`
- **Dependencies:** MRK-001 (Merkle root), PRF-003 (materialized counters), STO-002/STO-003 (metadata CF/DB)
- **Edge cases:**
  - Fresh store: height=0, tip_hash=zero hash, state_root=empty tree root, timestamp=0.
  - After full rollback: same as fresh store.
