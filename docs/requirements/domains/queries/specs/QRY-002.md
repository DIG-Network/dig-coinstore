# QRY-002: Puzzle Hash Queries

> **Authoritative requirement:** [QRY-002](../NORMATIVE.md#QRY-002)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

---

## Summary

Puzzle hash queries retrieve all coin records associated with a given puzzle hash (or set of puzzle hashes). These are the primary wallet-facing queries, as wallets track coins by their puzzle hash (derived from the wallet's public key). The queries support filtering by spent status and confirmed height range.

This mirrors Chia L1's `get_coin_records_by_puzzle_hash()` and `get_coin_records_by_puzzle_hashes()` methods.

---

## Specification

### Function Signatures

```rust
impl CoinStore {
    pub fn get_coin_records_by_puzzle_hash(
        &self,
        puzzle_hash: &Bytes32,
        include_spent: bool,
        start_height: Option<u32>,
        end_height: Option<u32>,
    ) -> Result<Vec<CoinRecord>, CoinStoreError>;

    pub fn get_coin_records_by_puzzle_hashes(
        &self,
        puzzle_hashes: &[Bytes32],
        include_spent: bool,
        start_height: Option<u32>,
        end_height: Option<u32>,
    ) -> Result<Vec<CoinRecord>, CoinStoreError>;
}
```

### Behavior

- If `include_spent` is `false`, only unspent coins are returned (uses `unspent_by_puzzle_hash` index per PRF-004).
- If `include_spent` is `true`, all coins (spent and unspent) are returned (uses `coin_by_puzzle_hash` index).
- `start_height` and `end_height` filter by `confirmed_height` (inclusive on both ends).
- `None` for either height bound means unbounded in that direction.

---

## Acceptance Criteria

- [ ] Returns all coins matching the puzzle hash.
- [ ] `include_spent=false` excludes spent coins.
- [ ] `include_spent=true` includes both spent and unspent coins.
- [ ] Height range filtering works correctly with both bounds, one bound, or no bounds.
- [ ] Batch version handles multiple puzzle hashes with deduplication.
- [ ] Empty puzzle_hashes slice returns empty result.

---

## Implementation Notes

- **Primary file:** `src/queries.rs`
- **Dependencies:** STO-002/STO-003 (puzzle hash indices), PRF-004 (unspent-only index)
- **Edge cases:**
  - Puzzle hash with no matching coins returns empty vector.
  - `start_height > end_height` returns empty vector.
