# QRY-006: Lightweight CoinState Queries

> **Authoritative requirement:** [QRY-006](../NORMATIVE.md#QRY-006)

> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

CoinState queries return a lightweight representation of coin state suitable for wallet sync protocols. Unlike full `CoinRecord` queries, `CoinState` contains only the essential fields needed for sync: the coin itself, its created height, and its spent height (if spent). This reduces bandwidth and processing overhead for sync operations.

This mirrors Chia L1's `get_coin_states_by_ids()` and related methods used by the wallet sync protocol.

## Specification

### Function Signatures

```rust
impl CoinStore {
    /// SPEC Section 3.8
    pub fn get_coin_states_by_ids(
        &self,
        include_spent: bool,
        coin_ids: &[Bytes32],
        min_height: Option<u32>,
        max_height: Option<u32>,
        max_items: Option<usize>,
    ) -> Result<Vec<CoinState>, CoinStoreError>;

    /// SPEC Section 3.6
    pub fn get_coin_states_by_puzzle_hashes(
        &self,
        include_spent: bool,
        puzzle_hashes: &[Bytes32],
        min_height: Option<u32>,
        max_items: Option<usize>,
    ) -> Result<Vec<CoinState>, CoinStoreError>;
}
```

### CoinState Struct

```rust
pub struct CoinState {
    pub coin: Coin,
    pub created_height: Option<u32>,
    pub spent_height: Option<u32>,
}
```

### Behavior

- `get_coin_states_by_ids()` (SPEC Section 3.8): Returns CoinState for each found coin ID. Missing IDs are silently skipped. If `include_spent` is `false`, only unspent coins are returned. `min_height` and `max_height` filter by `confirmed_height` (inclusive). `max_items` limits the number of returned results; if `None`, no limit is applied.
- `get_coin_states_by_puzzle_hashes()` (SPEC Section 3.6): Returns CoinState for all coins matching any of the provided puzzle hashes. If `include_spent` is `false`, only unspent coins are returned. `min_height` filters by `confirmed_height` (inclusive lower bound). `max_items` limits the number of returned results; if `None`, no limit is applied.

## Acceptance Criteria

- [ ] Returns `CoinState` with correct `coin`, `created_height`, and `spent_height` fields.
- [ ] `spent_height` is `Some(h)` for spent coins, `None` for unspent.
- [ ] `created_height` is `Some(h)` for all coins.
- [ ] Missing IDs are silently skipped.
- [ ] `include_spent=false` excludes spent coins from results.
- [ ] `include_spent=true` includes both spent and unspent coins.
- [ ] `min_height` / `max_height` filtering by `confirmed_height` works correctly.
- [ ] `max_items` limits the result set size when provided.
