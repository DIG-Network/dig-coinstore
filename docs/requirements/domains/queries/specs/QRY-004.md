## Summary

Parent ID queries retrieve all coin records that were created by a given parent coin (or set of parent coins). This is used for tracing coin lineage, CPFP (child-pays-for-parent) scenarios, and singleton tracking. The queries support filtering by spent status and height range.

This mirrors Chia L1's `get_coin_records_by_parent_ids()` method.

---

## Specification

### Function Signature

```rust
impl CoinStore {
    pub fn get_coin_records_by_parent_ids(
        &self,
        parent_ids: &[Bytes32],
        include_spent: bool,
        start_height: Option<u32>,
        end_height: Option<u32>,
    ) -> Result<Vec<CoinRecord>, CoinStoreError>;
}
```

### Behavior

- Returns all coins where `parent_coin_info` matches any of the provided parent IDs.
- Uses the `coin_by_parent` index for efficient lookup.
- If `include_spent` is `false`, only unspent coins are returned.
- `start_height` and `end_height` filter by `confirmed_height` (inclusive).

## Acceptance Criteria

- [ ] `include_spent=false` excludes spent coins.
- [ ] Height range filtering works correctly.
- [ ] Multiple parent IDs are handled in a single query.
- [ ] Empty parent_ids slice returns empty result.

## Implementation Notes

- **Primary file:** `src/queries.rs`
- **Dependencies:** STO-002/STO-003 (parent index)
- **Edge cases:**
  - Parent ID with no children returns empty vector.
  - A parent may have multiple children (e.g., a coin that creates multiple outputs).
