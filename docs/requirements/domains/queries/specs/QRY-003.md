# QRY-003: Height-Based Addition and Removal Queries

> **Authoritative requirement:** [QRY-003](../NORMATIVE.md#QRY-003)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

---

## Summary

Height-based queries retrieve coins by the block height at which they were created (added) or spent (removed). These are essential for block-level state tracking, chain sync, and rollback operations. The removal query has a special case: height 0 always returns an empty result since no coins can be spent at the genesis height.

This mirrors Chia L1's `get_coins_added_at_height()` and `get_coins_removed_at_height()` methods.

---

## Specification

### Function Signatures

```rust
impl CoinStore {
    pub fn get_coins_added_at_height(&self, height: u32) -> Result<Vec<CoinRecord>, CoinStoreError>;
    pub fn get_coins_removed_at_height(&self, height: u32) -> Result<Vec<CoinRecord>, CoinStoreError>;
}
```

### Behavior

- `get_coins_added_at_height()`: Returns all coins where `confirmed_height == height`. Uses the `coin_by_confirmed_height` index.
- `get_coins_removed_at_height()`: Returns all coins where `spent_height == height`. Uses the `coin_by_spent_height` index. If `height == 0`, MUST return an empty vector immediately (no coins are spent at genesis).

---

## Acceptance Criteria

- [ ] `get_coins_added_at_height()` returns all coins confirmed at the given height.
- [ ] `get_coins_removed_at_height()` returns all coins spent at the given height.
- [ ] `get_coins_removed_at_height(0)` returns an empty vector.
- [ ] Heights with no matching coins return empty vectors.
- [ ] Results include full `CoinRecord` with all fields populated.

---

## Implementation Notes

- **Primary file:** `src/queries.rs`
- **Dependencies:** STO-002/STO-003 (height indices), PRF-007 (height-partitioned indices)
- **Edge cases:**
  - Height beyond current tip: returns empty vector (not an error).
  - Genesis block (height 0) may have added coins but never removed coins.
