# CON-001: CoinStore Thread Safety (Send + Sync)

> **Authoritative requirement:** [CON-001](../NORMATIVE.md#CON-001)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md](../../../../resources/SPEC.md) — Section 11.1

---

## Summary

`CoinStore` MUST be both `Send` and `Sync`, meaning it can be safely transferred between threads and shared across threads via references. This is a foundational requirement for any concurrent usage: without `Send + Sync`, the coinstate cannot be placed behind an `Arc` for shared ownership across async tasks or thread pool workers.

All public methods on `CoinStore` must be callable from any thread without the caller needing to provide external synchronization. The internal locking strategy (CON-002) handles all necessary synchronization.

## Specification

### Trait Bounds

```rust
// CoinStore must satisfy these auto-trait bounds:
// - Send: safe to transfer ownership to another thread
// - Sync: safe to share &CoinStore across threads

// Compile-time assertion (placed in tests or a static_assertions block):
fn _assert_send_sync() {
    fn assert_send<T: Send>() {}
    fn assert_sync<T: Sync>() {}
    assert_send::<CoinStore>();
    assert_sync::<CoinStore>();
}
```

### Requirements for Inner Types

For `CoinStore` to be `Send + Sync`, all of its fields must also be `Send + Sync`. This constrains the choice of inner types:

| Field | Requirement | Notes |
|-------|-------------|-------|
| Storage backend (RocksDB/LMDB) | `Send + Sync` | RocksDB `DB` is `Send + Sync`. LMDB `Environment` is `Send + Sync`. |
| RwLock | `Send + Sync` | `parking_lot::RwLock<T>` is `Send + Sync` when `T: Send`. |
| HashSet (unspent set) | `Send + Sync` | Standard `HashSet` is `Send + Sync` when contents are. |
| LRU cache | `Send + Sync` | Must verify the chosen LRU crate's type is `Send + Sync`. |

### Public Method Contract

Every public method on `CoinStore` MUST be safe to call from any thread:
- Query methods: `get_coin_record()`, `is_unspent()`, `get_coins_by_puzzle_hash()`, etc.
- Mutation methods: `apply_block()`, `rollback()`.
- Metadata methods: `height()`, `tip_hash()`, `unspent_count()`, etc.

No method requires the caller to hold an external lock or coordinate with other threads.

## Acceptance Criteria

- [ ] `CoinStore` implements `Send`.
- [ ] `CoinStore` implements `Sync`.
- [ ] Compile-time static assertion verifies `Send + Sync`.
- [ ] All public methods are callable from spawned threads without external synchronization.
- [ ] `CoinStore` can be wrapped in `Arc<CoinStore>` and shared across threads.

---

## Implementation Notes

- **Primary file:** `src/coin_store.rs`
- **Dependencies:** All inner types must be `Send + Sync`
- **Edge cases:**
  - If any inner type is not `Send + Sync` (e.g., a raw pointer-based cache), it must be wrapped in an appropriate synchronization primitive.
  - `Rc<T>` is not `Send`; use `Arc<T>` instead.
  - `Cell<T>` and `RefCell<T>` are not `Sync`; use `Mutex<T>` or `RwLock<T>` instead.
