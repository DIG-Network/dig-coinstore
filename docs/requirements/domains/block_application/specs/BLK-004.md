## Summary

Every non-genesis block must include a minimum number of coinbase (reward) coins. This assertion catches malformed blocks before any state mutation occurs. The genesis block is special: it has no rewards because there is no prior block to reward.

This is adopted from Chia L1's `CoinStore.new_block()` ([`coin_store.py:138-141`](https://github.com/Chia-Network/chia-blockchain/blob/6e7a4954edccd8ab83fcacf938cfc42ddfcad7f2/chia/full_node/coin_store.py#L138)):

```python
if height > 0:
    assert len(reward_coins) >= 2  # farmer reward + pool reward
else:
    assert len(reward_coins) == 0
```

In Chia, the two mandatory reward coins are the farmer reward and the pool reward. dig-coinstore enforces the same minimum via `MIN_REWARD_COINS_PER_BLOCK = 2`.

## Specification

### Validation Logic

```rust
const MIN_REWARD_COINS_PER_BLOCK: usize = 2;

if block.height == 0 {
    if !block.coinbase_coins.is_empty() {
        return Err(CoinStoreError::InvalidRewardCount {
            height: block.height,
            expected_min: 0,
            got: block.coinbase_coins.len(),
        });
    }
} else {
    if block.coinbase_coins.len() < MIN_REWARD_COINS_PER_BLOCK {
        return Err(CoinStoreError::InvalidRewardCount {
            height: block.height,
            expected_min: MIN_REWARD_COINS_PER_BLOCK,
            got: block.coinbase_coins.len(),
        });
    }
}
```

---

## Acceptance Criteria

- [ ] Genesis block (height 0) with 0 coinbase coins is accepted.
- [ ] Genesis block (height 0) with >0 coinbase coins is rejected.
- [ ] Non-genesis block with >= 2 coinbase coins is accepted.
- [ ] Non-genesis block with 1 coinbase coin is rejected.
- [ ] Non-genesis block with 0 coinbase coins is rejected.
- [ ] Non-genesis block with exactly 2 coinbase coins is accepted (boundary case).
- [ ] Non-genesis block with >2 coinbase coins is accepted.

## Implementation Notes

- **Primary file:** `src/coin_store.rs` (or `src/block_application.rs`)
- **Dependencies:** BLK-001 (entry point), BLK-002/003 (run before this check)
- **Edge cases:**
  - Height 1 is the first block that requires coinbase coins.
  - Blocks may have more than 2 coinbase coins (e.g., additional fee coins).
