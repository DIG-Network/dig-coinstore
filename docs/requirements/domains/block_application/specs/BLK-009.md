# BLK-009: State Root Verification

## Summary

After all coin mutations (insertions and spend markings) are complete and the Merkle tree has been updated, the coinstate may verify that the computed state root matches an expected value provided in the block data. This is an optional but critical integrity check: if the block producer claims a particular state root, the coinstate verifies that applying the block's state changes actually produces that root.

A mismatch indicates either:
- The block producer computed the state root incorrectly.
- The local coinstate has diverged from the expected chain state.
- The `BlockData` was corrupted or tampered with.

In all cases, the block must be rejected and the mutations rolled back.

---

## Specification

### Verification Logic

```rust
if let Some(expected_root) = block.expected_state_root {
    let computed_root = self.merkle_tree.root();
    if computed_root != expected_root {
        return Err(CoinStoreError::StateRootMismatch {
            expected: expected_root,
            computed: computed_root,
        });
    }
}
```

### Error Type

```rust
CoinStoreError::StateRootMismatch {
    expected: Bytes32,   // block.expected_state_root
    computed: Bytes32,   // self.merkle_tree.root()
}
```

### Optional Nature

`BlockData.expected_state_root` is `Option<Bytes32>`:
- **`Some(root)`**: Verification is performed. Mismatch is a hard error.
- **`None`**: Verification is skipped. The block is accepted based on the other validation checks alone. This mode is useful during initial sync or when the block producer does not include a state root commitment.

### Ordering

This check runs in Phase 2, after:
1. BLK-007 (coin insertion)
2. BLK-008 (spend marking)
3. Merkle tree batch update

It runs before the chain tip update, so a failed state root check prevents the tip from advancing.

---

## Acceptance Criteria

- [ ] If `expected_state_root` is `Some(root)` and computed root matches, the check passes.
- [ ] If `expected_state_root` is `Some(root)` and computed root does not match, returns `CoinStoreError::StateRootMismatch`.
- [ ] If `expected_state_root` is `None`, no verification occurs and the block proceeds.
- [ ] The computed root is taken from the Merkle tree after all mutations for this block are applied.
- [ ] A failed state root check rolls back all mutations for this block.

---

## Implementation Notes

- **Primary file:** `src/coin_store.rs` (or `src/block_application.rs`)
- **Dependencies:** BLK-007 (coin insertion), BLK-008 (spend marking), Merkle tree update
- **Edge cases:**
  - Genesis block: The state root after applying genesis is the Merkle root of the initial coinbase coins (or empty tree root if genesis has no coins).
  - Empty block (no additions, no removals): The state root should not change from the prior block's root.
  - Merkle tree implementation: The exact root computation depends on the Merkle tree implementation (sparse Merkle tree, as specified in the merkle domain requirements).

| `test_state_root_match` | Integration | Apply block with correct expected_state_root | Succeeds |
