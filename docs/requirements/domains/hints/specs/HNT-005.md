# HNT-005: Rollback Cleanup

> **Authoritative requirement:** [HNT-005](../NORMATIVE.md#HNT-005)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

---

## Summary

When a chain rollback deletes coins that were confirmed after the target height, all hints associated with those deleted coin IDs must also be removed from both the forward and reverse indices. This ensures the hint store does not accumulate orphaned entries that reference coins no longer in the coin store.

This is an improvement over Chia L1, which leaves orphaned hint rows in the database after rollback. Chia's `coin_store.py:rollback()` does not call any hint cleanup, so the `hints` table grows monotonically. Our implementation explicitly cleans up hints to maintain index consistency and prevent stale query results.

---

## Specification

### Function Signature

```rust
impl HintStore {
    /// Removes all hints associated with the given coin IDs from both indices.
    pub fn remove_hints_for_coins(
        &mut self,
        coin_ids: &[CoinId],
    ) -> Result<u64, HintError>;
}
```

### Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `coin_ids` | `&[CoinId]` | The coin IDs being deleted during rollback. |

### Return Value

- **`Ok(count)`** -- The number of hint entries removed (across both indices, counted once per logical `(coin_id, hint)` pair).
- **`Err(HintError)`** -- Storage error during cleanup.

### Behavior

1. For each `coin_id` in `coin_ids`:
   a. Prefix-scan the forward index with `coin_id` to find all hints associated with this coin.
   b. For each `(coin_id, hint)` entry found:
      - Delete the forward index entry: key = `coin_id ++ hint`.
      - Delete the reverse index entry: key = `hint ++ coin_id`.
   c. Increment the removal counter.
2. Update the materialized hint count (decrement by total removed).
3. Return the total number of hint pairs removed.

### Integration with Rollback

This function MUST be called during `rollback_to_block()` (RBK-002) after identifying coins to delete but before or during the coin deletion transaction. Both coin deletion and hint cleanup MUST occur within the same write transaction to maintain atomicity.

### Ordering

```
rollback_to_block(target_height)
  1. Identify coins confirmed after target_height
  2. Save pre-deletion snapshots to modified_coins
  3. remove_hints_for_coins(coin_ids_to_delete)  <-- HNT-005
  4. Delete coins from all indices (RBK-002)
  5. Un-spend coins spent after target_height (RBK-003)
  ...
```

## Acceptance Criteria

- [ ] All hints for rolled-back coins are deleted from the forward index.
- [ ] All hints for rolled-back coins are deleted from the reverse index.
- [ ] No orphaned entries remain in either index after rollback.
- [ ] Querying a deleted coin's hints returns empty results after rollback.
- [ ] Querying a hint that only pointed to deleted coins returns empty results after rollback.
- [ ] Hints for coins NOT being deleted are unaffected.
- [ ] Hint count is decremented correctly after rollback cleanup.
- [ ] Cleanup and coin deletion occur within the same transaction (atomicity).

## Implementation Notes

- **Primary file:** `src/hints.rs` (cleanup function), `src/rollback.rs` (integration point)
- **Dependencies:** HNT-003 (both indices must be cleaned), RBK-002 (coin deletion triggers this)
- **Performance:**
  - Batch the prefix scans: for each coin_id, scan once and collect all hints before deleting.
  - If rolling back many blocks with many coins, this could involve significant I/O. Consider batching deletes.
- **Chia divergence:** This is an explicit improvement. Chia L1 does NOT clean up hints during rollback, leading to an ever-growing hints table with stale entries. Our approach ensures hint queries always return accurate results.
- **Edge cases:**
  - Coin with no hints: prefix scan returns empty, no deletions needed.
  - Coin whose hint is shared with other non-deleted coins: only the `(coin_id, hint)` forward entry and the `(hint, coin_id)` reverse entry are deleted. Other coins' entries for the same hint remain intact.
  - Empty `coin_ids` slice: no-op, returns 0.
