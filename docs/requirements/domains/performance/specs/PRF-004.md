# PRF-004: Unspent-Only Puzzle Hash Index

> **Authoritative requirement:** [PRF-004](../NORMATIVE.md#PRF-004)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

A dedicated `unspent_by_puzzle_hash` column family indexes only currently-unspent coins by their puzzle hash. This is the primary index for wallet queries ("show me all unspent coins for this puzzle hash"), which are by far the most common query pattern.

By maintaining a separate index that only contains unspent coins, query performance is dramatically better than scanning a full index (which includes all historical spent coins) and filtering. On a mature chain with hundreds of millions of spent coins but only ~20 million unspent, this index is an order of magnitude smaller.

The index is maintained transactionally: entries are inserted when coins are created, deleted when coins are spent, and re-inserted when spends are rolled back.

---

## Specification

### Column Family

```
unspent_by_puzzle_hash: puzzle_hash||coin_id -> () (empty value, existence-only)
```

### Key Format

The key is a concatenation of the puzzle hash (32 bytes) and the coin ID (32 bytes), yielding a 64-byte compound key. The value is empty -- the index is existence-only. To retrieve full records, the caller uses the coin ID to look up the `coin_records` CF.

### Operations

| Operation | Trigger | Action |
|-----------|---------|--------|
| Insert | Coin creation (apply_block addition) | `put(puzzle_hash \|\| coin_id, [])` |
| Delete | Coin spend (apply_block removal) | `delete(puzzle_hash \|\| coin_id)` |
| Re-insert | Rollback of a spend | `put(puzzle_hash \|\| coin_id, [])` |
| Delete (rollback) | Rollback of a creation | `delete(puzzle_hash \|\| coin_id)` |

### Query

To find all unspent coins for a puzzle hash:
```rust
prefix_scan(unspent_by_puzzle_hash, puzzle_hash)
```
This returns all coin IDs with that puzzle hash that are currently unspent.

## Acceptance Criteria

- [ ] Dedicated `unspent_by_puzzle_hash` CF exists.
- [ ] Entry inserted on coin creation.
- [ ] Entry deleted on coin spend.
- [ ] Entry re-inserted on rollback of a spend.
- [ ] Entry deleted on rollback of a creation.
- [ ] Index contains only unspent coins (no spent coins).
- [ ] Prefix scan by puzzle hash returns all unspent coin IDs for that puzzle hash.
- [ ] Index is substantially smaller than a full puzzle hash index.

---

## Implementation Notes

- **Primary file:** `src/storage/rocksdb.rs` or `src/indices.rs`
- **Dependencies:** Storage layer (column family management), BLK-007/BLK-008 (index updates during apply_block), RBK-* (index updates during rollback)
- **Edge cases:**
  - Puzzle hash with no unspent coins: prefix scan returns empty.
  - Coin created and spent in the same block: insert then delete in the same WriteBatch (net effect: not in index).
  - Multiple coins with the same puzzle hash: all appear in the prefix scan.
