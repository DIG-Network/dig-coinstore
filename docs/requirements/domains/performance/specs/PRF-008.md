## Summary

The snapshot/restore persistence API provides the methods for saving and loading coinstate snapshots. This is the programmatic interface that supports the snapshot-based fast sync workflow described in PRF-006. The API manages snapshot lifecycle including creation, retrieval by height, retrieval of the latest snapshot, listing available snapshots, and automatic pruning of old snapshots.

---

## Specification

### API Methods

```rust
impl CoinStore {
    /// Save a snapshot of the current coinstate at the current height.
    pub fn save_snapshot(&self) -> Result<SnapshotInfo, CoinStoreError>;

    /// Load a snapshot at a specific height.
    pub fn load_snapshot(height: u64) -> Result<CoinStore, CoinStoreError>;

    /// Load the most recent available snapshot.
    pub fn load_latest_snapshot() -> Result<CoinStore, CoinStoreError>;

    /// List all available snapshot heights, sorted ascending.
    pub fn available_snapshot_heights() -> Result<Vec<u64>, CoinStoreError>;
}
```

### SnapshotInfo

```rust
struct SnapshotInfo {
    height: u64,
    merkle_root: Hash,
    timestamp: u64,
    unspent_count: u64,
    file_size_bytes: u64,
}
```

### Auto-Pruning

When the number of saved snapshots exceeds `max_snapshots`, the oldest snapshots MUST be automatically deleted. Pruning occurs after a successful `save_snapshot()`.

### Configuration

| Parameter | Default | Description |
|-----------|---------|-------------|
| `max_snapshots` | 5 | Maximum number of snapshots to retain |
| `snapshot_dir` | `./snapshots/` | Directory for snapshot files |

### Error Conditions

| Error | Condition |
|-------|-----------|
| `SnapshotNotFound` | `load_snapshot(height)` for a height with no snapshot |
| `NoSnapshotsAvailable` | `load_latest_snapshot()` when no snapshots exist |
| `SnapshotCorrupted` | Snapshot file fails integrity check |
| `StorageError` | I/O error during save/load |

---

## Acceptance Criteria

- [ ] `save_snapshot()` persists the current coinstate to a snapshot file.
- [ ] `load_snapshot(height)` restores a coinstate from the snapshot at the given height.
- [ ] `load_latest_snapshot()` restores from the most recent snapshot.
- [ ] `available_snapshot_heights()` lists all snapshot heights in ascending order.
- [ ] Old snapshots beyond `max_snapshots` are automatically pruned after save.
- [ ] `SnapshotNotFound` returned for missing height.
- [ ] `NoSnapshotsAvailable` returned when no snapshots exist.
- [ ] `SnapshotCorrupted` returned for tampered snapshot files.

---

## Implementation Notes

- **Primary file:** `src/snapshot.rs`
- **Dependencies:** PRF-006 (snapshot-based fast sync uses these methods), storage layer (serialization)
- **Edge cases:**
  - `max_snapshots = 0`: no snapshots retained (effectively disables snapshotting).
  - `save_snapshot()` called concurrently: should be serialized or use file-level locking.
  - Snapshot directory does not exist: create it on first save.
  - Disk full during save: return `StorageError`, do not leave partial snapshot files.
